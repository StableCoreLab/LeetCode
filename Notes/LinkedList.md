# Linked List

## 适用场景

- 需要频繁修改节点之间的连接关系
- 题目要求原地重排、翻转、合并链表
- 需要通过哑节点统一处理头节点变更

## 当前题目对应

### `23. Merge k Sorted Lists`

当前仓库里给了三种思路：

- `Solve`：顺序两两合并
  - 把答案先设为第一条链表
  - 之后依次和后面的链表做 `MergeTwo`
  - 实现直接，但当 `k` 较大时会有重复合并开销

- `Solve2`：分治合并
  - 用 `MergeRange(lists, left, right)` 递归拆分区间
  - 左右两半分别合并后，再做一次 `MergeTwo`
  - 本质上是把“多次顺序合并”优化成“平衡合并”

- `Solve3`：优先队列
  - 把每条链表当前头节点放进最小堆
  - 每次取出最小节点接到结果后面
  - 如果该节点还有 `next`，继续入堆

其中核心基础操作都是合并两条有序链表：

```cpp
while (a && b) {
    if (a->val <= b->val) {
        tail->next = a;
        a = a->next;
    } else {
        tail->next = b;
        b = b->next;
    }
    tail = tail->next;
}
tail->next = a ? a : b;
```

### `25. Reverse Nodes in k-Group`

当前实现采用“分组切断 + 局部翻转 + 重新连接”：

1. 用哑节点 `dummy` 统一处理头节点变化
2. 先找到当前组的第 `k` 个节点，确认这一组长度足够
3. 记录：
   - 组前驱 `pPreGroupEnd`
   - 组头 `pGroupTail`
   - 下一组组头 `pNextGroupHead`
4. 暂时断开当前组尾部
5. 调用 `ReverseList` 翻转这一组
6. 把翻转后的新组头、旧组头、下一组重新接回去

这里有一个很关键的角色变化：

- 翻转前的组头，翻转后会变成组尾
- 所以它正好负责接回下一组

## 常见模型

- 哑节点 `dummy`
- 快慢/前后指针定位关键节点
- 局部翻转一段链表
- 归并多个有序链表

## 复杂度特点

### `Merge k Sorted Lists`

- 顺序两两合并：时间复杂度通常为 `O(kn)`，空间复杂度 `O(1)`
- 分治合并：时间复杂度 `O(n log k)`，递归栈空间 `O(log k)`
- 最小堆：时间复杂度 `O(n log k)`，额外空间 `O(k)`

这里 `n` 表示所有链表节点总数，`k` 表示链表条数。

### `Reverse Nodes in k-Group`

- 时间复杂度：`O(n)`
- 空间复杂度：`O(1)`

## 易错点

- 不使用哑节点时，头节点变化会很难处理
- 翻转一段链表前如果没有先保存后继节点，容易丢链
- `k` 组翻转时，剩余节点不足 `k` 个要保持原样
- 合并多链表时要先判断空链表输入
- 优先队列方案里，只有非空头节点才能入堆
