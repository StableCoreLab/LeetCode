# Stack

## 适用场景

- 需要维护“最近一个还没处理完的元素”
- 括号匹配、表达式处理
- 需要从右往左或从左往右维护边界关系
- 适合做“匹配失败就回退到上一个状态”的问题

## 当前题目对应

### `32. Longest Valid Parentheses`

当前仓库已经有题目目录、头文件和测试骨架，但 `Solve` 还没有补完具体实现。

这题最常见的栈做法是：

- 栈里存下标，不直接存字符
- 先压入一个哨兵下标 `-1`
- 遇到 `'('` 就入栈当前位置
- 遇到 `')'` 就先弹栈
  - 如果弹完栈空了，说明当前右括号无法匹配，把它的下标压回去作为新的边界
  - 如果栈不空，当前合法长度就是 `i - st.top()`

典型框架：

```cpp
stack<int> st;
st.push(-1);

for (int i = 0; i < s.size(); ++i) {
    if (s[i] == '(') {
        st.push(i);
    } else {
        st.pop();
        if (st.empty()) {
            st.push(i);
        } else {
            ans = max(ans, i - st.top());
        }
    }
}
```

## 常见模型

- 括号匹配栈
- 单调栈
- 用栈保存下标而不是值
- 哨兵简化边界处理

## 复杂度特点

- 大多数栈题都能做到线性扫描
- 每个元素通常只会入栈、出栈各一次
- 常见复杂度是时间 `O(n)`、空间 `O(n)`

## 易错点

- 括号题里如果栈空了，没有及时重置边界，下一个长度会算错
- 存字符不存下标时，通常没法直接计算区间长度
- 忘记放哨兵时，前缀刚好合法的情况容易漏掉
- 写 `pop` 前要先确认当前逻辑不会对空栈操作
