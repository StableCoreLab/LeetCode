# Binary Search

## 适用场景

- 搜索有序数组中的目标值
- 搜索答案
- 在满足单调性的条件下找边界
- 不直接找元素，而是找“分割位置”

## 当前题目对应

### `4. Median of Two Sorted Arrays`

核心不是在数组里找某个数，而是在较短数组中二分一个切分点：

- 令 `i` 是 `nums1` 的切分位置
- 令 `j` 是 `nums2` 的切分位置
- 保证左半部分总长度等于右半部分，或只多一个元素
- 满足：

```text
nums1[i - 1] <= nums2[j]
nums2[j - 1] <= nums1[i]
```

找到这个合法切分后：

- 总长度为奇数，中位数是左半部分最大值
- 总长度为偶数，中位数是左半部分最大值和右半部分最小值的平均值

## 模板思路

```cpp
int left = 0, right = n;
while (left <= right) {
    int mid = left + (right - left) / 2;
    if (满足条件) {
        return 答案;
    } else if (mid 偏大) {
        right = mid - 1;
    } else {
        left = mid + 1;
    }
}
```

## 这题的关键技巧

- 始终在更短的数组上二分，避免切分位置越界。
- 用 `INT_MIN` 和 `INT_MAX` 处理切分点在边界时的比较。
- 把问题转成“左边全部小于等于右边”的分治判断。

## 复杂度

- 时间复杂度：`O(log(min(m, n)))`
- 空间复杂度：`O(1)`

## 易错点

- 没有先交换数组，导致在长数组上二分，边界更难处理。
- 奇偶长度公式写错。
- `mid` 对应的是切分位置，不是元素下标。
- 边界切分时访问 `i - 1`、`j - 1` 越界。
